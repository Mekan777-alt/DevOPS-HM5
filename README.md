# Сервис управления задачами с использованием RabbitMQ и Redis

Этот сервис предназначен для добавления, управления и мониторинга задач с использованием RabbitMQ для передачи сообщений и Redis для хранения и управления очередями задач.

## Описание API

### Добавление задачи
Метод POST `/api/v1/add-task` добавляет новую задачу в систему.

#### Параметры запроса
- `task`: Объект типа `Task`, содержащий следующие поля:
  - `name`: Название задачи.
  - `description`: Описание задачи.
  - `params`: Параметры задачи в формате ключ-значение.

#### Ответ
Возвращает JSON с сообщением о успешном добавлении задачи и её уникальным идентификатором.

### Получение статистики
Метод GET `/api/v1/get-stats` возвращает количество успешно выполненных задач.

#### Ответ
Возвращает JSON с количеством выполненных задач.

### Получение очереди задач
Метод GET `/api/v1/get-task-queue` возвращает текущую очередь задач из Redis.

#### Ответ
Возвращает JSON с массивом задач в очереди.

### Обновление задачи
Метод PUT `/api/v1/update-task/{taskID}` обновляет задачу в очереди Redis по её идентификатору `taskID`.

#### Параметры запроса
- `task`: Объект типа `Task`, содержащий поля для обновления задачи.

#### Ответ
Возвращает JSON с сообщением о успешном обновлении задачи и массивом обновлённых задач.

### Удаление задачи
Метод DELETE `/api/v1/delete-task/{taskID}` удаляет задачу из очереди по её идентификатору `taskID`.

#### Ответ
Возвращает JSON с сообщением о успешном удалении задачи и массивом оставшихся задач.

## Технологии
Сервис использует следующие технологии:
- FastAPI для создания веб-приложения и обработки запросов.
- RabbitMQ для асинхронной обработки сообщений.
- Redis для хранения и управления очередями задач.
